# CrisisBridge: Operational Workflow

This document details the end-to-end user journeys for both guests in distress and hotel responders, as well as the underlying automated safety mechanisms.

---

### 👤 1. The Guest Journey (Signaling Distress)
**Goal:** Zero-friction emergency notification in under 2 seconds.

1.  **Entry:** Guest scans a QR code or taps an NFC sticker. The **Signal Flare** web app opens instantly (No login/No app install).
2.  **Context (Optional):** Guest enters a brief description (e.g., *"My friend is having a seizure"*).
3.  **The Trigger:** Guest **presses and holds** the "MEDICAL" button for 0.5s.
    *   *Safety Logic:* The 0.5s hold prevents accidental triggers while remaining fast enough for true crises.
4.  **AI Interaction:** The system displays *"AI Triaging Emergency..."* while Google Gemini analyzes the input.
5.  **Intelligence Report:** Guest receives immediate AI-generated instructions (e.g., *"Help is on the way. Place the person on their side and clear the area."*) to maintain calm and provide life-saving advice.

---

### 🛡️ 2. The Responder Journey (Crisis Coordination)
**Goal:** Rapid response guided by actionable intelligence.

1.  **Active Monitoring:** Staff monitor the **Responder Ops Dashboard** at the security station.
2.  **Instant Alert:** A new flare causes the dashboard to pulse red with an audible alert.
3.  **AI Action Card:** The responder sees a Gemini-generated task card (e.g., *"Bring AED to Room 305 immediately. Call 911."*) instead of a generic alarm.
4.  **Acknowledgment:** Responder clicks **"Acknowledge"**.
    *   *Impact:* The Guest UI updates to show help is moving, and the **30-second escalation timer is stopped**.
5.  **Resolution:** Once the crisis is managed, the responder clicks **"Resolve"**.
6.  **Audit Trail:** A **Finalized Incident Report** is automatically generated by AI, summarizing the timeline, description, and actions taken for legal and insurance records.

---

### ⚙️ 3. The Automated Safety Net (System Logic)
**Goal:** Resilience against human error and technical failure.

*   **30-Second Escalation:** If an alert remains unacknowledged for 30 seconds, the system automatically triggers a **Critical Escalation Email** via SendGrid to senior management.
*   **Offline Fallback:** If the guest loses data connectivity, the UI displays an **Amber Alert Banner** with one-tap links to **Call 911** or send a **Cellular SMS Flare**.
*   **Background Sync:** Using Service Workers, alerts triggered during temporary network drops are queued and automatically delivered the moment connectivity is restored.
*   **Tamper-Resistant Logging:** Every state change (Triggered -> Acknowledged -> Resolved) is logged to a persistent database for accountability.

---

### 🏨 4. B2B Property Onboarding (Missing in Current Workflow)
**Goal:** Operationalize CrisisBridge for hotels at room/floor/property scale.

1.  **Property Setup:** Hotel admin creates Property -> Floors -> Rooms in a B2B admin console.
2.  **Room URL Generation:** System generates one signed URL per room (example: `https://help.hotel.com/?property=H100&room=305`).
3.  **Bulk QR Export:** Download printable sheets (PNG/PDF) grouped by floor/room with label text (Room ID + fallback short code).
4.  **NFC Sticker Provisioning:** Program NFC tags with the same room URL (`NDEF URL record`), then physically place stickers near the room entry/bedside panel.
5.  **Installation QA:** Staff scan/tap every room artifact and confirm room mapping before go-live.

**What is an NFC sticker?**  
An NFC sticker is a low-cost programmable tag. A guest taps a phone to it, and the phone opens the encoded emergency URL without camera scanning.

### 🔎 6. Required Validation for Emergency Readiness

1.  **QR/NFC Ops Test:** Randomly sample rooms and verify scan/tap opens the correct room context.
2.  **Guest Impact Test:** Trigger alert -> acknowledge in responder UI -> verify guest status changes in real-time.
3.  **Offline + Sync Test:** Submit alert while offline -> reconnect -> verify queued request is replayed and persisted.
4.  **Audit Integrity Test:** Confirm Triggered/Acknowledged/Resolved are append-only in persistent storage (immutable event log).
